# UI Library

Create a standard UI library so shared components have the same design and feel
across the product.

## Implementation

- Use a headless React component library under the `packages` folder.
- Use React Storybook to document components and view them in the browser.
- The same UI package should be consumed by the front-end app and Storybook.
- Components should use Tailwind CSS, shadcn-compatible conventions, and Vite,
  and should be compatible with TanStack Start.
- Once setup is complete, create basic UI components like button, tabs, input,
  and dropdown.
- Use Vitest for testing. For this spec, only set up testing; do not write
  actual tests.

## Styling Guideline

- Use standard practices.
- If anything is not working, pause and decide mutually.
